# 卡尔曼滤波器使用指南

## 概述

卡尔曼滤波器是一种最优估计算法，用于在存在噪声的情况下估计动态系统的状态。在鼠标控制中，它用于预测目标的运动轨迹，提高跟踪精度。

相比原有的简单导数预测器，卡尔曼滤波器具有以下优势：
- **噪声抑制**：显式建模测量噪声和过程噪声
- **最优估计**：基于数学理论的最优预测
- **自适应能力**：可根据检测置信度动态调整

## 配置参数说明

### 1. 启用卡尔曼滤波器 (useKalmanFilter)

**类型**: 布尔值 (开/关)  
**默认值**: 开启

**效果**:
- **开启**: 使用卡尔曼滤波器进行目标运动预测
- **关闭**: 使用原有的简单导数预测器 (DerivativePredictor)

**建议**:
- 大多数场景建议开启，特别是目标移动较快或检测噪声较大时
- 如果目标基本静止且检测非常稳定，可以关闭以获得更低延迟

---

### 2. 过程噪声 (kalmanProcessNoise)

**类型**: 浮点数  
**范围**: 0.001 ~ 0.1  
**默认值**: 0.01

**物理意义**: 表示系统模型不确定性的大小。值越大，表示对"目标做匀速运动"这个假设的信任度越低。

**调整效果**:

| 值 | 效果 |
|---|---|
| 较小 (0.001~0.005) | 更平滑的跟踪，响应较慢，适合目标运动稳定的情况 |
| 中等 (0.01~0.02) | 平衡平滑度和响应速度，适合大多数场景 |
| 较大 (0.05~0.1) | 更敏感的跟踪，响应更快，但可能引入更多抖动 |

**调整建议**:
- **目标运动稳定** (如慢速移动): 使用较小值 (0.005~0.01)
- **目标运动剧烈** (如快速变向): 使用较大值 (0.02~0.05)
- **检测噪声大**: 适当增大，让滤波器更依赖预测而非测量

---

### 3. 测量噪声 (kalmanMeasurementNoise)

**类型**: 浮点数  
**范围**: 0.1 ~ 10.0  
**默认值**: 1.0

**物理意义**: 表示检测结果的不确定性。值越大，表示对检测结果(目标位置)的信任度越低。

**调整效果**:

| 值 | 效果 |
|---|---|
| 较小 (0.1~0.5) | 更信任检测结果，预测占比较小，适合高精度检测 |
| 中等 (1.0~2.0) | 平衡检测和预测，适合大多数场景 |
| 较大 (5.0~10.0) | 更依赖预测，检测影响较小，适合检测不稳定的情况 |

**调整建议**:
- **检测精度高** (高置信度、稳定): 使用较小值 (0.5~1.0)
- **检测精度低** (低置信度、抖动): 使用较大值 (2.0~5.0)
- **目标频繁被遮挡**: 使用较大值，让滤波器在丢失检测时依靠预测

---

### 4. 置信度缩放 (kalmanConfidenceScale)

**类型**: 浮点数  
**范围**: 0.1 ~ 5.0  
**默认值**: 1.0

**物理意义**: 用于根据检测置信度动态调整测量噪声的缩放因子。

**工作原理**:
```
实际测量噪声 = 基础测量噪声 / (置信度缩放 × 检测置信度 + 0.1)
```

**调整效果**:

| 值 | 效果 |
|---|---|
| 较小 (0.1~0.5) | 置信度对滤波器影响较小，各检测权重差异不大 |
| 中等 (1.0) | 正常缩放，高置信度检测更受重视 |
| 较大 (2.0~5.0) | 置信度影响显著，高置信度检测主导滤波器更新 |

**调整建议**:
- **置信度可靠** (模型输出的置信度准确反映检测质量): 使用较大值 (1.5~2.0)
- **置信度不可靠**: 使用较小值 (0.5~1.0)
- **需要平衡高低置信度检测**: 使用默认值 1.0

## 参数组合建议

### 场景1: 快速移动目标 (如FPS游戏中的跑动敌人)

```
过程噪声: 0.03
测量噪声: 1.5
置信度缩放: 1.2
```
- 较大的过程噪声允许滤波器快速响应目标变向
- 适中的测量噪声平衡检测和预测
- 置信度缩放让高质量检测更有影响力

### 场景2: 稳定移动目标 (如慢速移动的敌人)

```
过程噪声: 0.005
测量噪声: 0.8
置信度缩放: 1.0
```
- 较小的过程噪声提供更平滑的跟踪
- 较小的测量噪声更信任检测结果
- 标准置信度缩放

### 场景3: 检测噪声大的场景

```
过程噪声: 0.02
测量噪声: 3.0
置信度缩放: 1.5
```
- 较大的测量噪声降低检测抖动的影响
- 适中的过程噪声保持一定的响应速度
- 置信度缩放区分好坏检测

### 场景4: 目标频繁遮挡/丢失

```
过程噪声: 0.01
测量噪声: 5.0
置信度缩放: 2.0
```
- 很大的测量噪声让滤波器主要依赖预测
- 较小的过程噪声保持预测稳定性
- 高置信度缩放确保好检测仍能影响滤波器

## 调试技巧

### 1. 观察跟踪效果

- **跟踪滞后**: 减小过程噪声或测量噪声
- **跟踪抖动**: 增大过程噪声或测量噪声
- **丢失目标后恢复慢**: 减小测量噪声或增大置信度缩放

### 2. 参数调整顺序

1. 首先调整**测量噪声**，找到检测和预测的平衡点
2. 然后调整**过程噪声**，优化响应速度和平滑度
3. 最后调整**置信度缩放**，微调高低置信度检测的影响

### 3. 对比测试

- 可以临时关闭卡尔曼滤波器，使用原有预测器对比效果
- 观察在目标快速变向、检测丢失恢复等场景下的差异

## 技术细节

### 状态向量

卡尔曼滤波器使用4维状态向量：
```
X = [x, y, vx, vy]^T
```
- x, y: 目标位置 (像素)
- vx, vy: 目标速度 (像素/秒)

### 系统模型

采用**恒定速度模型** (Constant Velocity Model)：
```
x(k+1) = x(k) + vx(k) × dt
y(k+1) = y(k) + vy(k) × dt
vx(k+1) = vx(k) + 噪声
vy(k+1) = vy(k) + 噪声
```

### 观测模型

只观测位置 (不直接观测速度)：
```
z_x = x + 测量噪声
z_y = y + 测量噪声
```

### 算法流程

每帧执行：
1. **预测步骤**: 根据上一状态预测当前状态
2. **更新步骤**: 融合预测和检测结果，得到最优估计
3. **输出**: 使用估计的状态进行目标预测

## 常见问题

**Q: 卡尔曼滤波器会增加延迟吗？**  
A: 不会。卡尔曼滤波器的计算复杂度为O(1)，每帧处理时间小于0.1ms，不会造成可感知的延迟。

**Q: 为什么有时候跟踪会滞后？**  
A: 可能是过程噪声或测量噪声设置过小，导致滤波器过于平滑。尝试增大这两个参数。

**Q: 检测置信度低时应该怎么办？**  
A: 增大测量噪声，让滤波器更依赖预测而非低质量的检测。

**Q: 卡尔曼滤波器和原有预测器的主要区别？**  
A: 原有预测器使用简单的指数平滑，卡尔曼滤波器使用最优估计理论，显式处理噪声，通常能提供更准确的预测。
